Verdict

Approve.

Well-structured fix for a real operational issue. The core reconciliation logic is correct — key format matches between config slugs and outage records, active-outage definition is consistent with existing queries, and audit logging is preserved. Tests cover the important scenarios. The e2e reload-detection rework (timestamp → monotonic counter) is a solid reliability improvement. Only nits below, nothing blocking.

What this PR does

  • Adds ResolveActiveOutagesForMissingSubComponents on *DBOutageManager: on startup and on each config hot-reload, fetches all active outages, compares their component/sub-component slug pairs against the current config, and sets EndTime on any that no longer exist — going through UpdateOutage so audit logs are recorded.
  • Wires the resolver into main.go: runs once at startup, then registers a configManager.OnUpdate callback for subsequent reloads.
  • Changes NewDBOutageManager return type from the OutageManager interface to the concrete *DBOutageManager so the new method is callable.
  • Adds reloadCount atomic.Uint64 to the config manager and moves the "Config reloaded successfully" log to after all OnUpdate callbacks complete, including the count in the log fields.
  • Reworks e2e config-reload detection from --since-time log parsing to --tail + monotonic reload_count comparison, eliminating clock-skew sensitivity.

Things I checked and found correct

  • Key format consistency: configuredSubComponentKeys uses c.Slug/sub.Slug. Outage records store slugs in ComponentName/SubComponentName (confirmed via component_monitoring_report.go:162-163, absent_report.go:153-154, handlers.go:236-237). Match is correct.
  • Active-outage definition: GetAllActiveOutages uses end_time IS NULL OR end_time > now(), matching the existing GetActiveOutagesForComponent query.
  • Audit trail: Resolution goes through m.UpdateOutage(o, resolverUser) which records in outage_audit_logs with user "dashboard-config-reload".
  • Nil safety: Nil config returns empty allowed set (resolves everything). Nil component pointers in the slice are skipped.
  • Atomics: reloadCount uses atomic.Uint64, incremented outside the mutex after callbacks — correct ordering.
  • E2e timing: Poll interval is 10s in e2e (--config-update-poll-interval 10s), well within the 90s test timeout.
  • Security: No auth bypass, no injection vectors. Audit trail maintained for automated resolutions.
  • Error handling: Individual outage-update failures are logged and skipped (partial progress preferred over all-or-nothing). The listing error is returned. Appropriate strategy.
